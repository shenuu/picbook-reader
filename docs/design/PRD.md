## 1. 产品概述

### 1.1 背景

随着家庭教育意识的不断提升，绘本阅读已成为 2–6 岁幼儿早期语言发展的重要方式。然而，家长普遍面临以下痛点：

- **时间碎片化**：职场家长无法随时陪读，幼儿独自翻阅时无人讲解文字；
- **认字门槛高**：幼儿尚不识字，看到图文并茂的绘本却无法自主阅读；
- **纸质绘本昂贵**：正版绘本价格不菲，家长希望延长单本绘本的使用价值；
- **市场现状**：现有听书类 App 大多需要购买数字版内容，无法服务于家中已有的实体绘本。

**绘本朗读助手**旨在通过拍照识字 + AI 朗读的方式，让幼儿能够自主"读"手边任意一本实体绘本，降低亲子阅读的门槛，同时减轻家长的陪读负担。

### 1.2 目标用户

| 用户角色 | 描述 | 核心诉求 |
|---|---|---|
| **主要用户：幼儿家长** | 有 2–6 岁子女的父母或祖父母，使用微信，有一定智能手机操作能力 | 让孩子能够自主听绘本朗读，减少陪读时间成本 |
| **间接用户：2–6 岁幼儿** | 处于语言敏感期，喜欢重复听故事，注意力持续时间短（5–15 分钟） | 好听的声音、有趣的朗读体验，能自主操作 |
| **潜在用户：幼儿园教师** | 用于课堂辅助朗读，一次性为多本绘本建立缓存 | 批量处理效率，稳定的课堂使用体验 |

### 1.3 核心价值主张

> **"拍一拍，绘本开口说话"**  
> 用手机摄像头扫描任意实体绘本页面，3 秒内即可获得儿童友好的 AI 朗读，让每一本纸质绘本都能"活"起来。

**核心价值维度：**

- 🎯 **零门槛**：无需购买数字版内容，家中任意绘本均可使用；
- 🔊 **儿童友好**：专为幼儿设计的儿童音色 TTS，语调亲切、节奏适中；
- 📡 **离线可用**：已读过的页面自动缓存，网络不稳定时仍可继续朗读；
- 🚀 **极速响应**：P90 响应时间 ≤ 3 秒，符合幼儿注意力特点；
- 🛡️ **安全可靠**：无需注册账号，保护儿童隐私，家长无后顾之忧。

## 2. 功能需求

### 2.1 功能模块总览

```
绘本朗读助手
├── F1 · 拍照识字（OCR）
├── F2 · TTS 朗读
│   ├── F2.1 · 音色选择（3种）
│   └── F2.2 · 播放控制
├── F3 · 缓存管理（LRU 20页）
└── F4 · 离线降级
```

---

### 2.1 F1 · 拍照识字（OCR）

#### 功能描述
用户通过小程序内置相机或从相册选取绘本页面照片，系统自动对全图进行 OCR 识别，提取页面上的全部文字内容。

#### 详细需求

| 需求编号 | 需求描述 | 优先级 |
|---|---|---|
| F1-01 | 提供「拍照」按钮，调用设备摄像头拍摄绘本页面 | P0 |
| F1-02 | 提供「从相册选取」入口，支持从本地相册导入图片 | P0 |
| F1-03 | 采用**自动全图识别**模式，无需用户手动框选文字区域 | P0 |
| F1-04 | 识别结果按原文布局顺序（从上至下、从左至右）排列 | P1 |
| F1-05 | 识别过程中显示加载动画，告知用户当前状态 | P1 |
| F1-06 | 识别失败时展示友好错误提示，并提供重拍入口 | P0 |
| F1-07 | 支持识别简体中文（主）、数字、基础英文字母 | P0 |
| F1-08 | 图片在上传前进行本地压缩（长边最大 1920px），节省流量 | P1 |

#### 交互说明
- 拍照界面提供辅助取景框，提示用户将绘本页面对齐；
- 拍摄完成后自动进入识别流程，无需手动确认；
- 识别完成后自动跳转朗读界面并开始播放。

---

### 2.2 F2 · TTS 朗读

#### 功能描述
将 OCR 识别得到的文字，通过 TTS 引擎合成为语音并播放。

#### F2.1 · 音色选择（3种）

| 音色编号 | 音色名称 | 适用场景 | 说明 |
|---|---|---|---|
| V01 | 🧒 **小明**（儿童男声） | 日常故事朗读 | 活泼明亮，语速适中 |
| V02 | 👧 **小美**（儿童女声） | 公主/童话类绘本 | 甜美温柔，节奏舒缓 |
| V03 | 👩 **暖妈**（成人女声） | 科普/知识类绘本 | 亲切稳重，表达清晰 |

| 需求编号 | 需求描述 | 优先级 |
|---|---|---|
| F2-01 | 提供音色选择入口（支持朗读前选择，朗读中切换） | P0 |
| F2-02 | 默认音色为「小明」（儿童男声） | P1 |
| F2-03 | 音色选择以图文卡片形式展示，提供试听功能 | P1 |
| F2-04 | 用户选择的音色跨会话持久保存 | P2 |

#### F2.2 · 播放控制

| 需求编号 | 需求描述 | 优先级 |
|---|---|---|
| F2-05 | 支持**播放 / 暂停**切换 | P0 |
| F2-06 | 支持**重新播放**（从头开始） | P0 |
| F2-07 | 朗读时在界面上同步**高亮当前朗读文字** | P1 |
| F2-08 | 支持**语速调节**（0.8× / 1.0× / 1.2× 三挡） | P2 |
| F2-09 | 朗读结束后自动停止，并展示「再读一遍」按钮 | P1 |
| F2-10 | 支持后台播放（用户切换到其他 App 时继续朗读） | P2 |

### 2.3 F3 · LRU 缓存（20页）

#### 功能描述
对已识别并朗读过的绘本页面，在本地维护一个容量为 20 页的 LRU（最近最少使用）缓存，存储 OCR 文字结果及对应 TTS 音频文件，避免重复调用网络接口。

#### 详细需求

| 需求编号 | 需求描述 | 优先级 |
|---|---|---|
| F3-01 | 每次成功识别后，将图片哈希、OCR 文字、TTS 音频缓存至本地 | P0 |
| F3-02 | 缓存容量上限为 **20 页**，超出时自动淘汰最近最少使用的页面 | P0 |
| F3-03 | 命中缓存时，直接加载本地音频，无需重新调用 OCR/TTS 接口 | P0 |
| F3-04 | 在设置页提供「清除缓存」功能，展示当前缓存占用大小 | P1 |
| F3-05 | 缓存采用图片内容哈希（而非文件名）作为键，保证准确命中 | P1 |
| F3-06 | 缓存数据持久化存储，App 重启后仍可命中历史缓存 | P0 |

#### 缓存策略说明
```
新图片输入
    ↓
计算图片哈希 → 查询本地缓存
    ↓                ↓
  未命中            命中
    ↓                ↓
调用OCR+TTS     直接播放本地音频
    ↓
存入缓存（若已满，淘汰LRU项）
```

---

### 2.4 F4 · 离线降级

#### 功能描述
当设备处于弱网或无网络环境时，系统优先使用本地缓存提供服务；若无缓存，则给出明确提示，避免用户长时间等待。

#### 详细需求

| 需求编号 | 需求描述 | 优先级 |
|---|---|---|
| F4-01 | 启动时检测网络状态，离线时在界面顶部展示「离线模式」标识 | P0 |
| F4-02 | 离线状态下，已缓存页面可正常朗读，体验与在线一致 | P0 |
| F4-03 | 离线状态下拍摄新页面时，给出提示「当前无网络，请先联网识别新页面」 | P0 |
| F4-04 | 网络恢复后，自动解除离线限制，无需用户手动刷新 | P1 |
| F4-05 | 弱网（请求超时 >5秒）时，主动提示用户并提供「继续等待 / 取消」选项 | P1 |

## 3. 非功能需求

### 3.1 性能需求

| 指标 | 要求 | 说明 |
|---|---|---|
| **端到端响应时间** | P90 ≤ 3 秒 | 从用户拍照完成 → 开始播放语音的全链路时间 |
| **缓存命中响应时间** | P90 ≤ 0.5 秒 | 命中本地缓存时的加载时间 |
| **OCR 识别时间** | P90 ≤ 1.5 秒 | 服务端 OCR 接口响应时间 |
| **TTS 合成时间** | P90 ≤ 1 秒 | 首包音频数据返回时间 |
| **App 冷启动时间** | ≤ 2 秒 | 从点击小程序图标到首屏可交互 |
| **内存占用** | ≤ 100 MB | 运行时峰值内存，避免系统强杀 |

### 3.2 包体积要求

| 包类型 | 限制 | 策略 |
|---|---|---|
| **主包** | ≤ 2 MB | 仅包含启动页、拍照页、朗读页核心逻辑 |
| **分包（可选）** | ≤ 8 MB / 个 | 设置页、帮助文档、音色资源按需分包加载 |
| **总包体积** | ≤ 20 MB | 微信小程序平台限制 |

> ⚠️ **关键约束**：主包必须控制在 2 MB 以内，超出将影响小程序审核通过及用户下载体验。

### 3.3 兼容性要求

| 维度 | 要求 |
|---|---|
| **操作系统** | iOS 13+ / Android 8.0+ |
| **微信版本** | ≥ 8.0.20 |
| **设备分辨率** | 适配 375px – 428px 宽度区间（主流机型） |
| **摄像头** | 后置摄像头分辨率 ≥ 800万像素 |
| **网络环境** | 支持 4G / 5G / Wi-Fi；离线模式支持缓存访问 |

### 3.4 可靠性与可用性

| 指标 | 要求 |
|---|---|
| **服务可用性** | ≥ 99.5%（月度统计） |
| **OCR 识别准确率** | ≥ 95%（标准绘本字体、正常光线条件下） |
| **TTS 合成成功率** | ≥ 99%（文字长度 ≤ 500 字） |
| **崩溃率** | ≤ 0.1%（每次启动） |

### 3.5 安全与隐私

| 要求 | 说明 |
|---|---|
| 图片不落地存储 | 上传至服务端的图片仅用于 OCR 识别，处理完成后立即删除，不长期存储 |
| 无需账号注册 | 不收集用户姓名、手机号等个人身份信息 |
| 数据本地化 | 缓存数据仅存储在用户设备本地，不上传至云端 |
| HTTPS 传输 | 所有网络请求均通过 HTTPS 加密传输 |

## 4. 用户故事

### US-01 · 拍照识字朗读（核心场景）

**作为** 一位 3 岁孩子的妈妈，  
**我希望** 能用手机拍一下绘本页面，  
**以便** 让孩子独自听这页故事，不需要我时刻陪在旁边。

**验收条件：**
- 打开小程序后，首屏有明显的「拍照朗读」按钮；
- 拍照后 3 秒内（P90）开始播放朗读音频；
- 朗读音色为儿童友好型，语速适中；
- 朗读过程中，屏幕上同步显示识别出的文字。

---

### US-02 · 音色选择（个性化场景）

**作为** 一位喜欢公主故事的 5 岁女孩的爸爸，  
**我希望** 能给孩子选择一个温柔的女声音色，  
**以便** 让朗读体验更符合孩子的喜好，让她更愿意听。

**验收条件：**
- 朗读界面有「切换音色」入口；
- 展示 3 种音色选项，每种有名称、图标和试听按钮；
- 切换音色后，当前页面重新以新音色朗读；
- 下次使用时，记住上次选择的音色。

---

### US-03 · 重复听同一页（离线缓存场景）

**作为** 一位经常带孩子乘地铁的家长，  
**我希望** 孩子喜欢的那几页故事在没有网络时也能反复播放，  
**以便** 在地铁里打发时间，不担心流量消耗。

**验收条件：**
- 已经拍照过的页面，再次拍同一页时直接命中缓存，0.5 秒内开始播放；
- 在飞行模式下，缓存页面可以正常朗读；
- 界面顶部有「离线模式」标识提示用户当前状态；
- 缓存最多保存 20 页，超出后自动淘汰最旧的页面。

---

### US-04 · 弱网提示（容错场景）

**作为** 一位在农村老家探亲的家长，网络信号不稳定，  
**我希望** 网络差的时候 App 能给我明确提示，而不是一直转圈，  
**以便** 我知道是网络问题，而不是 App 出了 bug。

**验收条件：**
- OCR 请求超过 5 秒未返回，弹出提示「网络较慢，继续等待还是取消？」；
- 无网络状态下尝试拍新页面，立即提示「当前无网络，无法识别新页面」；
- 提示语言简洁，有重试按钮；
- 网络恢复后可以继续正常使用，无需重启 App。

---

### US-05 · 清除缓存（存储管理场景）

**作为** 一位手机存储空间有限的家长，  
**我希望** 能看到缓存占用了多少空间，并可以手动清除，  
**以便** 在手机存储不足时释放空间，不让这个 App 占用太多资源。

**验收条件：**
- 设置页展示「当前缓存大小：X MB（共 Y 页）」；
- 点击「清除缓存」后二次确认，确认后缓存清空；
- 清除完成后，缓存大小显示为 0；
- 清除后仍可正常使用拍照朗读功能（需联网）。

---

### US-06 · 首次使用引导（新用户场景）

**作为** 第一次使用这个小程序的家长，  
**我希望** 看到简单的使用引导，  
**以便** 快速了解如何操作，不需要阅读复杂的说明。

**验收条件：**
- 首次启动显示不超过 3 步的使用引导；
- 引导图文并茂，操作指引直观；
- 引导可跳过，不强制观看；
- 引导结束后直接进入拍照界面。

---

### US-07 · 语速调节（进阶个性化场景）

**作为** 一位想让孩子跟读练习普通话的家长，  
**我希望** 能放慢朗读语速，  
**以便** 孩子有足够时间跟着一字一字地模仿发音。

**验收条件：**
- 朗读界面提供语速调节按钮（慢 0.8× / 正常 1.0× / 快 1.2×）；
- 切换语速后立即生效（当前句子完成后切换）；
- 语速偏好跨页面保留（本次会话内）。

## 5. 验收标准

### 5.1 功能验收标准

| 测试用例 | 期望结果 | 通过标准 |
|---|---|---|
| 标准绘本、正常光线下拍照 | 识别准确率 ≥ 95% | 字符错误率（CER）≤ 5% |

| 纯图片页面（无文字） | 提示「本页没有文字」 | 不崩溃，友好提示 |
| 网络超时（模拟） | 弹出超时提示 | 5 秒内弹出，有重试按钮 |

| 测试用例 | 期望结果 | 通过标准 |
|---|---|---|
| 100 字以内文本 | 完整朗读，无截断 | 全文朗读完整 |
| 包含数字、英文的混合文本 | 正确读出数字和英文 | 无跳读、乱读 |
| 切换音色 | 新音色立即生效 | 切换后首次播放使用新音色 |
| 朗读中暂停，再继续 | 从暂停位置继续 | 不从头重播 |

#### 缓存与离线

| 测试用例 | 期望结果 | 通过标准 |
|---|---|---|
| 重复拍摄同一页面 | 命中缓存，快速播放 | ≤ 0.5 秒开始播放 |
| 缓存 20 页后再添加第 21 页 | 淘汰最旧页面 | 缓存中仍为 20 页 |
| 飞行模式下访问缓存页 | 正常播放 | 不报错，完整播放 |
| 飞行模式下拍摄新页面 | 提示无网络 | 2 秒内弹出提示 |

### 5.2 性能验收标准

| 测试场景 | 指标 | 通过标准 |

| 命中缓存，加载播放 | P90 响应时间 | ≤ 0.5 秒 |
| 小程序冷启动 | 首屏可交互时间 | ≤ 2 秒 |
| 主包体积 | 编译后体积 | ≤ 2 MB |

## 6. 版本规划

### Phase 1 · MVP（目标上线：T+8 周）

**目标**：验证核心价值，上架微信小程序平台

**交付功能：**
- ✅ F1 · 拍照识字（全图自动识别）
- ✅ F2.1 · 3 种音色选择
- ✅ F2.2 · 基础播放控制（播放/暂停/重播）
- ✅ F3 · LRU 缓存（20页）
- ✅ F4 · 离线降级（缓存命中 + 无网提示）
- ✅ 首次使用引导（US-06）
- ✅ 基础设置页（清除缓存）

**成功指标：**
- 7 日留存率 ≥ 30%
- 人均每日朗读次数 ≥ 3 次
