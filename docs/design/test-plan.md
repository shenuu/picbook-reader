## 1. 测试策略总览

### 1.1 项目概述

| 属性 | 说明 |
|------|------|
| 产品名称 | 绘本朗读助手 |
| 平台 | 微信小程序（原生） |
| 目标用户 | 2–6 岁幼儿的家长 |
| 核心流程 | 拍照 → 腾讯云 OCR 识别 → 讯飞 TTS WebSocket 流式朗读 |
| 性能目标 | P90 端到端响应时间 ≤ 3 秒 |
| 缓存策略 | LRU，最大 20 页 |

### 1.2 测试范围

#### 在范围内（In Scope）

- 所有主要页面的 UI 功能：首页、拍照引导、OCR 加载、识别结果、识别失败、TTS 播放器、声音切换、离线 Toast、缓存管理、离线降级页
- 核心业务流程：拍照→OCR→TTS 完整链路
- LRU 缓存的命中、淘汰与联动逻辑
- 网络异常降级（断网/弱网）
- 兼容性（iOS/Android 机型 + 微信版本）
- 性能（P90 ≤ 3 秒、内存、包大小）
- 安全（API 密钥保护、用户隐私合规）

#### 不在范围内（Out of Scope）

- 微信平台底层能力测试（如相机系统权限）
- 腾讯云 OCR / 讯飞 TTS 服务端的压力测试
- 小程序审核流程自动化

## 3. 性能测试方案

### 3.1 P90 ≤ 3 秒目标拆解

完整链路：**拍照快门 → OCR 识别 → TTS 首包播放**

| 阶段 | 描述 | 目标耗时（P90） | 测量起点 | 测量终点 |
|------|------|----------------|----------|----------|
| **图片压缩 & 上传** | 拍照后图片压缩并上传至 OCR 接口 | ≤ 400ms | 快门触发 | OCR 请求发出 |
| **OCR 识别** | 腾讯云 OCR 处理并返回文本 | ≤ 1500ms | 请求发出 | 收到响应体 |
| **TTS 请求建连** | 讯飞 WebSocket 建连 + 首帧音频返回 | ≤ 800ms | 发送 TTS 请求 | 首帧音频包到达 |
| **音频解码 & 播放** | 首帧解码并开始出声 | ≤ 300ms | 首帧到达 | 用户听到声音 |
| **总计（P90）** | | **≤ 3000ms** | 快门触发 | 用户听到声音 |

> 缓存命中场景目标：P90 ≤ 300ms（跳过 OCR + TTS 重新合成）

### 3.2 性能测试方法

#### 3.2.1 端到端耗时测量

```javascript
// 在关键节点打点（示例）
const perf = {
  shutterAt: 0,
  ocrRequestAt: 0,
  ocrResponseAt: 0,
  ttsConnectAt: 0,
  ttsFirstFrameAt: 0,
  audioPlayAt: 0,
};
// 计算各阶段 delta，上报至日志服务
```

- **工具**：微信开发者工具 Performance 面板 + 自定义打点上报
- **方式**：在代码中插入 `Date.now()` 打点，通过 vConsole 或日志服务收集
- **样本量**：每个场景至少 **50 次**有效数据，取 P50/P90/P99

#### 3.2.2 弱网性能测试

| 网络条件 | 模拟方式 | 通过标准 |
|----------|----------|----------|
| 4G 良好（50Mbps） | 默认 | P90 ≤ 3s |
| 4G 一般（10Mbps） | Charles 限速 | P90 ≤ 4s |
| 3G（2Mbps） | Charles 限速 | P90 ≤ 8s，有加载状态 |
| 弱网（500Kbps） | Charles 限速 | 有降级提示，不崩溃 |

#### 3.2.3 内存与包体积测试

| 指标 | 工具 | 通过标准 |
|------|------|----------|
| 小程序包体积（主包） | 微信开发者工具 | ≤ 2MB |
| 小程序包体积（分包） | 微信开发者工具 | 每个分包 ≤ 2MB |
| 内存占用（空闲时） | 开发者工具 Memory 面板 | ≤ 100MB |
| 内存占用（播放中） | 开发者工具 Memory 面板 | ≤ 200MB |
| 内存泄漏 | 连续操作 20 次后检测 | 无明显增长趋势 |

#### 3.2.4 首屏加载性能

| 指标 | 通过标准 |
|------|----------|
| 首页首屏渲染完成时间 | ≤ 2s（冷启动） |
| 首页首屏渲染完成时间 | ≤ 800ms（热启动） |
| FCP（首次内容绘制） | ≤ 1.5s |

### 3.3 性能测试通过标准汇总

| 指标 | P50 | P90 | P99 | 失败条件 |
|------|-----|-----|-----|----------|
| 全链路耗时（4G） | ≤ 2s | **≤ 3s** | ≤ 5s | P90 > 3s 则阻断上线 |
| OCR 单阶段耗时 | ≤ 1s | ≤ 1.5s | ≤ 3s | P90 > 2s 触发告警 |
| TTS 首帧耗时 | ≤ 500ms | ≤ 800ms | ≤ 1.5s | P90 > 1s 触发告警 |
| 缓存命中响应 | ≤ 150ms | ≤ 300ms | ≤ 500ms | P90 > 500ms 为缺陷 |

### 10.2 缺陷严重程度（Severity）

| 等级 | 名称 | 触发条件 | 示例 |
|------|------|----------|------|

| **S3 - 一般** | Major | 功能有缺陷但可通过其他方式完成，或影响体验 | 识别结果偶发乱序；缓存命中率偏低 |
| **S4 - 轻微** | Minor | UI/UX 问题，不影响功能使用 | 按钮文字截断；动画卡顿一帧 |
