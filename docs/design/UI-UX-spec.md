## 1. 设计原则

### 1.1 核心理念

本产品的主要用户是**2–6 岁儿童的家长**，使用场景多为亲子共读时的辅助操作，可能在光线较暗的卧室、户外或移动状态中单手操作。设计须**化繁为简**，让家长一眼看懂、一步完成。

### 1.2 五大原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **简洁直觉（Clarity First）** | 每个页面只做一件事，核心操作区域面积 ≥ 60pt，避免多层嵌套菜单 |
| 2 | **容错友好（Forgiving UX）** | 提供明确的撤销/重试入口；OCR 识别失败后给出引导而非空白页 |
| 3 | **温暖亲子（Warm & Playful）** | 视觉语言采用圆润、柔和的插画风格；图标有轻微卡通感但不幼稚 |
| 4 | **离线优先（Offline Ready）** | 已缓存内容无感切换；离线状态用暖色调 Toast 提示而非冷色系错误弹窗 |
| 5 | **无障碍可读（Accessible）** | 正文最小 16px，关键按钮文字 ≥ 18px；色觉障碍友好，不仅依赖颜色传递状态 |

## 2. 颜色系统

### 2.1 主色（Primary）

| Token | 色值 | 使用场景 |
|-------|------|----------|
| `--color-primary` | `#FF7043` | CTA 按钮、高亮操作区、进度条 |
| `--color-primary-light` | `#FFCCBC` | 按钮 Hover/Press 背景、选中态浅底 |
| `--color-primary-dark` | `#E64A19` | 按钮按下态、强调文字 |

> 主色选用暖橙红，传递活力与亲子感，同时与白底对比度 ≥ 4.5:1（WCAG AA）。

### 2.2 辅色（Secondary）

| Token | 色值 | 使用场景 |
|-------|------|----------|
| `--color-secondary` | `#42A5F5` | 次要按钮、链接、标签栏激活态 |
| `--color-secondary-light` | `#BBDEFB` | 信息提示背景、卡片描边 |
| `--color-accent` | `#FFD54F` | 星星评分、勋章、引导高亮 |

### 2.3 中性色（Neutral）

| Token | 色值 | 使用场景 |
|-------|------|----------|
| `--color-bg-base` | `#FFFDF8` | 全局页面背景（暖白，避免纯白刺眼） |
| `--color-bg-card` | `#FFFFFF` | 卡片、面板背景 |
| `--color-bg-muted` | `#F5F5F5` | 禁用区域、分割线背景 |
| `--color-text-primary` | `#212121` | 正文标题 |
| `--color-text-secondary` | `#757575` | 副文本、说明文字 |
| `--color-text-disabled` | `#BDBDBD` | 禁用状态文字 |
| `--color-divider` | `#E0E0E0` | 分割线 |

### 2.4 语义色（Semantic）

| Token | 色值 | 使用场景 |
|-------|------|----------|
| `--color-success` | `#4CAF50` | 成功态、已缓存角标底色 |
| `--color-success-light` | `#C8E6C9` | 成功提示背景 |
| `--color-warning` | `#FF9800` | 警告文字、注意提示 |
| `--color-error` | `#F44336` | 错误态、识别失败提示 |
| `--color-info` | `#2196F3` | 普通信息 Toast |

### 2.5 特殊语义色

#### 离线橙色（Offline Warm）

```
背景色：#FFF3E8   ← 暖橙背景，非高饱和警告红
文字色：#E65100   ← 深橙文字，确保对比度
图标色：#FF9800   ← 橙色图标
边框色：#FFCC80   ← 柔和橙色边框
```

> **设计意图**：离线状态是正常使用场景（地铁、飞机），不应使用红色引起焦虑，暖橙色传递"一切正常，只是当前没网"的安心感。

#### 缓存绿色（Cache Green）

```
角标背景：#4CAF50
角标图标：白色云朵 ☁️（16×16pt）
角标尺寸：20×20pt，位于卡片右上角，偏移 (-6pt, -6pt)
```

## 3. 字体规范

### 3.1 字体族

```
--font-family-base: "PingFang SC", "Helvetica Neue", Arial, sans-serif;
--font-family-number: "DIN Alternate", "PingFang SC", monospace;
```

> 微信小程序优先使用系统字体，确保跨平台渲染一致性。

### 3.2 字号体系

| Token | 大小 | 行高 | 字重 | 使用场景 |
|-------|------|------|------|----------|
| `--text-xs` | `12px` | `1.4` | 400 | 角标、时间戳（≥12px，不低于此） |
| `--text-sm` | `14px` | `1.5` | 400 | 辅助说明、标签文字 |
| `--text-base` | `16px` | `1.6` | 400 | **最小可读字号**，正文段落 |
| `--text-md` | `18px` | `1.6` | 400 | 列表项、次要标题 |
| `--text-lg` | **`20px`** | `1.5` | 500 | **默认正文字号**，按钮文字、卡片标题 |
| `--text-xl` | `24px` | `1.4` | 600 | 页面主标题、大型 CTA |
| `--text-2xl` | `28px` | `1.3` | 700 | 首屏 Hero 标题 |
| `--text-3xl` | `32px` | `1.2` | 700 | 数字大字展示（如倒计时） |

> **规则**：
> - 正文段落默认使用 `--text-lg`（20px），确保家长在持机距离下无需眯眼阅读。
> - 任何情况下不得低于 `--text-base`（16px），角标例外（最低 12px）。
> - 需要突出重要信息时，优先加粗而非缩小字号。

### 3.3 字重规范

| 场景 | 字重 |
|------|------|
| 正文内容 | `400 Regular` |
| 副标题、按钮 | `500 Medium` |
| 页面标题、强调 | `600 SemiBold` |
| 首屏标题、数字 | `700 Bold` |

## 4. 间距与圆角规范

### 4.1 间距系统（基础单位 4pt）

```
--space-1:  4pt   ← 最小内边距，图标与文字间距
--space-2:  8pt   ← 紧凑布局内边距
--space-3: 12pt   ← 列表项内边距、小型组件内间距
--space-4: 16pt   ← 标准内边距（最常用）
--space-5: 20pt   ← 卡片内边距
--space-6: 24pt   ← 区块间距
--space-8: 32pt   ← 大区块间距、页面顶部留白
--space-10: 40pt  ← 页面底部安全区补偿
--space-12: 48pt  ← 首屏 Hero 区域上下留白
```

### 4.2 页面边距

```
页面水平边距：--space-4（16pt）
内容区最大宽度：375pt（以 iPhone 13 mini 为基准，自适应更大屏）
底部 TabBar 高度：56pt + 安全区高度
```

### 4.3 圆角系统

| Token | 值 | 使用场景 |
|-------|-----|----------|
| `--radius-sm` | `4pt` | 标签、角标、小型输入框 |
| `--radius-md` | `8pt` | 列表项、选项卡 |
| `--radius-lg` | `12pt` | 卡片、面板 |
| `--radius-xl` | `16pt` | 底部弹窗（Bottom Sheet） |
| `--radius-2xl` | `24pt` | 大型卡片、主要功能区块 |
| `--radius-full` | `9999pt` | 圆形按钮、Pill 标签、头像 |

> **设计意图**：圆角越大越温和，主操作区域（TTS 播放按钮等）使用 `--radius-full`，传递亲子友好感；信息卡片使用 `--radius-lg`，专业而不呆板。

## 5. 组件规范

### 5.1 TTS 播放按钮

**规格**：`60 × 60 pt`，圆形（`border-radius: 30pt`）

#### 状态定义

| 状态 | 图标 | 背景色 | 阴影 | 行为说明 |
|------|------|--------|------|----------|
| **未播放（Idle）** | ▶️ 播放三角，白色，32pt | `--color-primary`（#FF7043） | `0 4pt 12pt rgba(255,112,67,.35)` | 点击后进入加载中 |
| **加载中（Loading）** | 旋转圆环动画（Lottie），白色，24pt | `--color-primary-light`（#FFCCBC） | 无阴影 | 禁止重复点击（防抖），显示进度环 |
| **播放中（Playing）** | ⏸ 暂停双竖线，白色，28pt | `--color-primary`（#FF7043） | `0 4pt 16pt rgba(255,112,67,.5)` | 按钮轻微脉冲动画（scale 1.0→1.05，1.5s 循环） |
| **完成（Done）** | ✅ 对勾图标，白色，28pt | `--color-success`（#4CAF50） | `0 4pt 12pt rgba(76,175,80,.3)` | 2s 后自动恢复至未播放态；可再次点击重播 |

#### 点击反馈

```
按下：scale(0.92)，duration: 100ms，ease: ease-in
松开：scale(1.0)，duration: 200ms，ease: spring(stiffness:300)
```

#### 无障碍

```
aria-label 动态更新：
  Idle    → "点击朗读"
  Loading → "正在加载语音"
  Playing → "点击暂停"
  Done    → "朗读完成，点击重播"
```

### 5.3 已缓存标记（缓存绿色云朵角标）

#### 规格

```
尺寸：20 × 20 pt
形状：圆形底座，radius-full
背景色：#4CAF50（--color-success）
图标：白色云朵 SVG，14 × 10 pt，居中
位置：卡片右上角，偏移 transform: translate(6pt, -6pt)
```

#### 出现动画

```
进场：scale(0→1) + opacity(0→1)，duration: 300ms，ease: spring
消失：scale(1→0) + opacity(1→0)，duration: 200ms，ease: ease-in
```

#### 使用场景

- 绘本卡片右上角：表示该绘本已缓存可离线播放
- 缓存管理页列表项：每行末尾展示
- 搜索结果中：辅助标识可离线内容

#### 无障碍

```
图标配有隐藏文字：aria-label="已缓存，可离线使用"
```

### 5.4 离线 Toast

#### 规格

```
位置：页面顶部，StatusBar 下方，水平居中
宽度：屏幕宽度 - 32pt（左右各 16pt margin）
高度：自适应，最小 48pt
圆角：--radius-lg（12pt）
```

#### 视觉样式

```
背景色：#FFF3E8（--color-offline-bg）
边框：1pt solid #FFCC80
左侧图标：📶 无信号图标，20pt，颜色 #FF9800
文字：#E65100，字号 --text-md（18px），字重 500
右侧关闭按钮：✕，20pt，颜色 #E65100，点击区域 44×44pt
```

#### Toast 文案

| 场景 | 标题 | 副文字 |
|------|------|--------|
| 进入离线状态 | "当前处于离线模式" | "已缓存的绘本可正常朗读 📚" |
| 操作需要网络 | "此功能需要联网" | "请检查网络后重试" |
| 缓存内容可用 | "离线内容已就绪" | "共 X 本绘本可离线朗读" |

#### 行为

```
自动消失：普通提示 3s 后淡出；需要行动的提示不自动消失
进场动画：从顶部 -60pt 滑入，duration: 350ms，ease: cubic-bezier(0.22,1,0.36,1)
出场动画：向顶部 -60pt 滑出 + opacity→0，duration: 250ms，ease: ease-in
同屏限制：同一时刻最多显示 1 条 Toast（新的替换旧的）
```

## 6. 页面清单

### P01 — 首页（Home）

| 项目 | 说明 |
|------|------|
| **用途** | 产品入口，展示最近阅读绘本、快速拍照入口 |
| **关键元素** | 顶部搜索栏、Banner 轮播（推荐绘本）、最近朗读列表、底部 TabBar |
| **关键交互** | 点击搜索栏聚焦并跳转搜索页；长按绘本卡片出现快捷菜单（朗读/删除缓存）；下拉刷新同步云端书单 |
| **离线处理** | 离线时隐藏搜索栏联网提示，展示"离线模式"角标；仅展示已缓存绘本 |

---

### P02 — 拍照引导页（Camera Guide）

| 项目 | 说明 |
|------|------|
| **用途** | 引导家长拍摄绘本文字，提高 OCR 识别准确率 |
| **关键元素** | 取景框动画（四角高亮框）、3 条拍照技巧提示、拍照按钮（80pt 白圆圈）、相册选择入口 |
| **关键交互** | 拍照按钮防抖（1s 内只触发一次）；取景框内容稳定 1.5s 后自动高亮提示"画面稳定"；支持从相册选图 |
| **离线处理** | 拍照和 OCR 不依赖网络，此页无离线限制 |

---

### P03 — OCR 加载页（OCR Loading）

| 项目 | 说明 |
|------|------|
| **用途** | 等待 OCR 识别结果，避免用户焦虑 |
| **关键元素** | 卡通放大镜 Lottie 动画（放大镜来回扫描文字效果）、进度提示文案轮播、取消按钮 |
| **Lottie 动画规格** | 200 × 200pt，循环播放；动画文件路径 `/assets/lottie/ocr-scanning.json`；颜色主题跟随 `--color-primary` |
| **文案轮播** | 每 2s 切换：「正在识别文字…」→「努力辨认中…」→「马上就好啦！」 |
| **关键交互** | 超时 10s 未返回结果自动跳转识别失败页；点击取消返回拍照引导页 |

---

### P04 — 识别结果页（OCR Result）

| 项目 | 说明 |
|------|------|
| **用途** | 展示 OCR 识别出的文字，供家长确认并启动朗读 |
| **关键元素** | 识别文字大卡片（可滚动）、置信度指示（高/中/低三色星标）、"重新拍摄"按钮、"开始朗读"CTA 按钮 |
| **关键交互** | 文字卡片支持双指缩放调整字号；点击单段文字可手动编辑修正；"开始朗读"跳转 TTS 播放器页 |
| **编辑态** | 文字变为可编辑状态，键盘弹起时内容区上移，底部固定"完成编辑"按钮 |

---

### P05 — 识别失败页（OCR Error）

| 项目 | 说明 |
|------|------|
| **用途** | OCR 识别失败时提供友好反馈和引导 |
| **关键元素** | 卡通困惑表情插画、失败原因说明、两个操作按钮（重新拍摄 / 手动输入） |
| **失败原因分类** | 图片模糊、文字过小、非文字内容、网络超时（各有对应提示文案） |
| **关键交互** | "重新拍摄"返回 P02；"手动输入"弹出全屏文字输入框；3s 后如无操作展示"需要帮助？"提示 |

---

### P06 — TTS 播放器页（TTS Player）

| 项目 | 说明 |
|------|------|
| **用途** | 核心功能页，展示文字并同步朗读，支持跟读高亮 |
| **关键元素** | 绘本封面/配图（上半屏）、文字展示区（高亮跟读）、TTS 播放按钮（60pt）、进度条、声音切换入口 |
| **关键交互** | 朗读时当前句子高亮（背景 `--color-primary-light`）；点击任意句子从该处开始朗读；双击文字进入编辑；横屏时自动切换为沉浸朗读模式 |
| **离线处理** | 已缓存 TTS 音频直接播放；未缓存时显示离线 Toast 并禁用播放按钮 |

---

### P07 — 声音切换面板页（Voice Picker）

| 项目 | 说明 |
|------|------|
| **用途** | 让家长选择 TTS 朗读声音 |
| **关键元素** | Bottom Sheet 面板（见 5.2 组件规范）、三个声音选项卡、预览播放按钮、确认按钮 |
| **关键交互** | 点击选项立即高亮选中；点击 ▶ 播放 3s 预览片段（播放中图标变暂停）；下滑手势关闭面板；点击蒙层关闭 |
| **记忆功能** | 选择结果持久化到 `wx.setStorageSync`，下次打开默认回填 |

---

### P08 — 离线 Toast（Offline Toast）

| 项目 | 说明 |
|------|------|
| **用途** | 全局离线状态感知提示（非独立页面，为悬浮组件） |
| **关键元素** | 见 5.4 组件规范 |
| **触发时机** | 网络状态由联网变为断网时自动弹出；尝试需要网络的操作时手动触发 |
| **关键交互** | 关闭后不再自动弹出（直到网络恢复再断开）；点击 Toast 跳转离线降级页查看详情 |

---

### P09 — 离线降级页（Offline Mode）

## 7. 交互规范

### 7.1 防抖与节流

| 触发元素 | 类型 | 时长 | 说明 |
|----------|------|------|------|
| 拍照按钮 | 防抖（Debounce） | **1000ms** | 避免连拍，等待 1s 后才允许再次触发 |
| TTS 播放按钮 | 防抖 | **500ms** | 避免快速双击导致重复请求 |
| 声音切换确认 | 防抖 | **300ms** | 防止频繁切换声音请求 |
| 搜索输入框 | 防抖 | **400ms** | 停止输入 400ms 后触发搜索请求 |
| 列表滚动加载更多 | 节流（Throttle） | **200ms** | 滚动事件节流，避免频繁触发 |
| 网络状态检测 | 节流 | **1000ms** | 避免网络抖动引起 Toast 频繁闪烁 |

### 7.5 反馈与状态一致性原则

1. **即时反馈**：所有可点击元素在 `100ms` 内给出视觉反馈（颜色/缩放），让用户感知到点击有效。
2. **状态可见**：当前页面的数据状态（离线/联网/加载中/错误）始终在 UI 上有对应指示，不出现"空白等待"。
3. **可逆操作**：删除、清空缓存等破坏性操作必须提供撤销或二次确认；不可逆操作（如清空全部）使用红色按钮并需输入确认文字。
4. **错误可恢复**：每个错误状态页面必须包含至少一个明确的"下一步操作"按钮，不让用户陷入死胡同。
5. **一致性**：相同功能在不同页面使用相同的图标、颜色和手势，降低学习成本。
